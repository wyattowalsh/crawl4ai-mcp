## What is MCP-Crawl4AI?

MCP-Crawl4AI is a **high-performance server implementation** that connects the [Model Context Protocol (MCP)](https://modelcontextprotocol.ai) with advanced web crawling capabilities. It enables large language models like Claude to interact with the live web, retrieving real-time information and structured content from websites.

This server is designed to be:

- **Fast and efficient** - Optimized for quick response times with adaptive concurrency
- **Easy to integrate** - Compatible with any MCP client (Claude, Cursor, custom apps)
- **Reliable** - Robust error handling and recovery, with detailed logging
- **Flexible** - Multiple transport options, crawling configurations, and output formats
- **Developer-friendly** - Clear API, comprehensive documentation, and easy extensibility
- **Production-ready** - Scalable architecture suitable for individual and enterprise use

## Quick Start

Once installed, you can start the server with the `mcp-crawl4ai` command:

```bash
# Option 1: Default - Start with STDIO transport
# (Ideal for direct client integration like Claude Desktop)
mcp-crawl4ai

# Option 2: Start with HTTP transport on localhost port 8000
mcp-crawl4ai --transport http --host 127.0.0.1 --port 8000

# Option 3: Install browser dependencies (first time)
mcp-crawl4ai --setup

# Option 4: See all available options and help
mcp-crawl4ai --help
```

### Connecting an MCP Client

  <Tab>

    import json
from fastmcp import MCPClient
    
    # Connect to the server
    client = MCPClient("http://localhost:8000")  # For HTTP transport
    
    # List available tools
    tools = client.list_tools()
    print(f"Available tools: {[tool['name'] for tool in tools]}")
    
    # Call a tool
    result = client.call_tool(

## Architecture

MCP-Crawl4AI is built with a modular, type-safe architecture emphasizing performance, reliability, and extensibility.

### Core Components

- **Server Layer**: Implements the MCP v1 specification with transport options (STDIO, HTTP)
- **Core Engine**: High-performance asynchronous crawling with memory-adaptive concurrency
- **Crawler Lifecycle**: Shared `AsyncWebCrawler` managed through FastMCP lifespan context
- **Session & Artifacts**: Session registry and artifact store for crawl state and retrieval
- **Extraction Pipeline**: Processes raw web content into LLM-friendly formats
- **Resource Manager**: Handles MCP resources with persistent storage and retrieval

### Runtime Options

Use the built-in CLI options to select transport and bootstrap browser dependencies:

```bash
mcp-crawl4ai --help
mcp-crawl4ai --transport http --host 127.0.0.1 --port 8000
mcp-crawl4ai --setup
```

### Modern Python Standards

- **Type Safety**: Comprehensive typing with runtime validation via Pydantic v2
- **Async-First**: Built on asyncio with proper concurrency patterns
- **Testing**: Extensive pytest suite with property-based testing
- **Documentation**: Google-style docstrings with complete type annotations
- **Dependency Management**: Modern pyproject.toml with uv-managed dependency groups
